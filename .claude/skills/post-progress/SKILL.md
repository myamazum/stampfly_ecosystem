## X 進捗投稿スキル

$ARGUMENTS を解釈:

### コミット選択オプション（排他的）
- `-n <数>`: 直近N件のコミットをまとめる（デフォルト: 1）
- `--day <数>`: 直近N日間のコミットをまとめる（例: `--day 7`で1週間分）
- `--select`: **インタラクティブ選択**（直近20件から番号で選択）
- `--commits <ハッシュ>`: **ハッシュ指定**（カンマ区切り、例: `f3a6083,139df24`）
- `--range <範囲>`: **範囲指定**（例: `4c8c60c..f3a6083`）

### その他のオプション
- `-m "<メッセージ>"`: 自動生成メッセージの前に追加するユーザーメッセージ
- `--dry-run`: 投稿せずプレビューのみ
- `--style <スタイル>`: fun（デフォルト）/formal/emoji

**Note**:
- コミット選択オプションは排他的（1つのみ指定可能）
- `-m`を指定すると、指定したメッセージが自動生成メッセージの前に追加されます（置き換えではなく追加）

### 実行手順

1. **コミット情報の取得**
   新しい `generate_post.py` スクリプトを使用:
   ```bash
   python3 .claude/skills/post-progress/scripts/generate_post.py [オプション]
   ```

   **オプション別の動作:**
   - `-n <数>`: `git log -n <数>` で直近N件取得
   - `--day <数>`: `git log --since="<数> days ago"` で期間指定
   - `--select`: インタラクティブ選択（直近20件から番号で選択）
   - `--commits <ハッシュ>`: 指定ハッシュのみ取得
   - `--range <範囲>`: `git log <範囲>` で範囲指定

2. **投稿文の生成**
   コミットメッセージを元に、以下のスタイルで投稿文を生成:

   **共通ルール:**
   - 800-900文字程度を目安に詳細な技術情報を含める（旧制限280文字の約3倍）
   - ハッシュタグを2-3個追加
     - プロジェクト固有タグ: stampfly_ecosystem ディレクトリの場合は #StampFly を含める
     - 技術タグ + 分野タグ（例: #制御工学 #ドローン開発）
   - 構造化された情報提供を重視

   **スタイル別:**
   - `fun`（デフォルト）: ストーリー性と技術詳細を組み合わせた拡張スタイル

     **投稿構成（800-900文字）:**
     1. **冒頭**（1-3行）:
        - `-m` オプション指定時: ユーザーメッセージ + 空行 + 自動生成メッセージ
        - `-m` オプションなし: 自動生成メッセージのみ（課題や動機、期間を明示）
     2. **実装内容**（3-5項目）: 主要な成果を箇条書き（「・」で列挙、括弧で詳細）
     3. **詳細セクション**（カテゴリ別に整理）:
        - 「◆カテゴリ名: 説明」形式で複数カテゴリを展開
        - 各カテゴリで技術的な背景や数値を含める
        - コミット件数や具体的な変更内容を記載
     4. **締め**（1-2行）: 成果のインパクト + （絵文字1個のみ）
     5. **ハッシュタグ**: 2-3個（技術 + 分野）

     **詳細度のポイント:**
     - 技術用語は具体的に（ツール名、手法名、フレームワーク名）
     - 数値や規模感を入れる（「100+パターン」「400Hz」「54件のコミット」など）
     - カテゴリ分けで可読性を確保
     - 「なぜ」「どのように」「何が改善されたか」を含める

   - `formal`: 絵文字なし、ビジネスライクな報告形式
   - `emoji`: 絵文字多め、視覚的にインパクトのある表現

3. **プレビュー表示**
   生成した投稿文をユーザーに表示し、以下を確認:
   - 投稿内容（拡張版フォーマット）
   - 文字数（目安800-900文字）
   - カテゴリ別の詳細情報
   - 投稿するか確認を求める

   `--dry-run` の場合はここで終了

4. **X への投稿**
   承認されたら以下を実行:
   ```bash
   python3 .claude/skills/post-progress/scripts/post_to_x.py "<投稿文>"
   ```

### 使用例

```bash
# 基本的な使い方
/post-progress                # 最新1件のコミット
/post-progress -n 5          # 直近5件のコミット
/post-progress --day 7       # 直近7日間のコミット

# 🆕 インタラクティブ選択（推奨）
/post-progress --select
# → 直近20件を表示、番号で選択（例: 1,2,5 と入力）

# 🆕 特定のコミットを指定
/post-progress --commits f3a6083,139df24
# → 指定したハッシュのコミットのみ投稿

# 🆕 範囲指定
/post-progress --range 4c8c60c..f3a6083
# → 指定範囲内の全コミット

# プレビューのみ（投稿しない）
/post-progress --select --dry-run
/post-progress -n 3 --dry-run

# ユーザーメッセージ追加
/post-progress --select -m "今週の開発成果"
/post-progress -n 7 -m "年末年始の開発成果をまとめました。"

# スタイル変更
/post-progress -n 10 --style formal
/post-progress --select --style emoji
```

### 環境変数の設定

X API の認証情報を環境変数に設定してください:

```bash
# ~/.zshrc に追加
export X_API_KEY="your-api-key"
export X_API_SECRET="your-api-secret"
export X_ACCESS_TOKEN="your-access-token"
export X_ACCESS_TOKEN_SECRET="your-access-token-secret"
```

### エラーハンドリング

- Git リポジトリでない場合: エラーメッセージを表示
- コミットがない場合: エラーメッセージを表示
- 環境変数が未設定の場合: 設定方法を案内
- API エラーの場合: エラー内容を表示
